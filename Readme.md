# TransFile256: Консольное многопоточное приложение потоковой компрессии и межпоточного обмена данными

Многопоточная программная система на C++, реализующая классический паттерн **Producer-Consumer** (Производитель-Потребитель) в рамках единого адресного пространства под ОС Windows. Приложение эмулирует передачу данных через аппаратное ограничение физического уровня с использованием фиксированной **потокобезопасной очереди** размером **256 байт**, с поддержкой сквозной потоковой компрессии и декомпрессии данных «на лету».

## Ключевая архитектура и технологии
*   **Парадигма:** Многопоточность (POSIX Threads / `std::thread`).
*   **Синхронизация потоков:** Потокобезопасная очередь обмена (FIFO), синхронизированная с помощью примитивов `std::mutex` и `std::condition_variable` с использованием современных предикатов для исключения состояний гонки (Race Conditions) и ложных пробуждений (*Spurious Wakeups*).
*   **Асинхронное конфигурирование:** Передача метаданных расчета блоков между потоками через примитивы `std::promise` и `std::future`.
*   **Алгоритмы сжатия:** Интеграция библиотеки **zlib** (потоковый многопоточный режим `deflate` / `inflate`).
*   **Устойчивость (Robustness):** Корректная обработка критических граничных условий (*Edge Cases*), включая побайтовую обработку пустых файлов (0 байт) без падения системы и генерации исключений.

## Встроенная телеметрия и профилирование (Telemetry)
Система снабжена модулем `Measure` для сквозного профилирования производительности «на лету» без использования внешних тяжелых профайлеров. В конце работы функция 'PerfomanceStats()` выгружает в консоль следующие метрики:
*   `[TIME]` **Высокоточный бенчмарк:** Время обработки через `std::chrono::steady_clock` с точностью до миллисекунд.
*   `[PERF]` **Пропускная способность (Throughput):** Чистая скорость сквозной обработки потока в МБ/сек.
*   `[MEMORY]` **High-Water Mark:** Пиковый размер очереди (контроль переполнения ОЗУ).
*   `[THREAD]` **Статистика планировщика:** Количество защитных блокировок Потребителя (минимизация нагрузки на CPU).
*   `[STATS]`  **Статистика сжатия:** Процент сжатия, т.е. отношение общего количество сжатых байт к размеру исходного файла.
*   `[VERIFY]` **Побайтовый верификатор:** Контроль бинарного совпадения файлов бит-в-бит.

## Системные требования
*  **Операционная система:** Windows 10 / Windows 11.
*  **Среда разработки (IDE):** Microsoft Visual Studio 2022 (или новее) с установленным компонентом *«Разработка классических приложений на C++»*.
*  **Компилятор:** MSVC (Microsoft Visual C++) с поддержкой стандарта **ISO C++14** или **C++17**.
*  **Зависимости:** Библиотека компрессии **zlib** (скомпилированная под Windows / x64).


## Инструкция по запуску
Запуск исполняемого файла осуществляется из консоли Windows с передачей трех позиционных аргументов:

```cmd
TransFile_256.exe [source_file] [output_file] [compression_mode]
```

### Спецификация аргументов:
*   `[source_file]` — Путь к исходному файлу для передачи.
*   `[output_file]` — Путь для сохранения результирующего файла.
*   `[compression_mode]` — Режим обработки данных:
    *   `0` — Без сжатия (прямая фрагментация файла по 256 байт).
    *   `1` — Активация многопоточной компрессии `zlib`.
